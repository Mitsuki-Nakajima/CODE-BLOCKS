## OVERVIEW
### Description
CODE BLOCKS is a hands-on programming system designed to teach fundamental Python logic using physical 3D-printed blocks instead of typed code. Each block represents a programming structure (e.g., loops, conditionals, variables). When arranged physically, a Unity application interprets the configuration and visualizes how the program would execute. The goal was to explore whether programming concepts could be understood through tangible interaction rather than text alone.

### Motivation
Programming is traditionally text-based and abstract, which can make it difficult for beginners to grasp structural logic.
We asked:

> What if programming could be assembled physically before being written?

By mapping abstract control flow structures to physical objects, we aimed to create a more intuitive and accessible learning experience — especially for beginners and visually impaired users.

### My Role
I worked primarily on the software side as the lead Unity developer.
- Built the Unity application in C#
- Designed the logic system that translated block configurations into structured program execution
- Implemented visualization of control flow and execution behavior
- Helped define how physical block arrangements map to internal data structures

### System Design
The system consists of:
1. Physical 3D-printed blocks representing programming constructs  
2. A digital interpretation layer in Unity  
3. A visualization system that simulates program behavior  
Block configurations are converted into a structured internal representation. The application then simulates execution step-by-step, allowing users to see how control flow operates. The key challenge was designing a mapping between physical arrangement and logical hierarchy.

### Technical Focus
- Unity (C#)
- Control flow modeling
- Structured logic representation
- Human–computer interaction design
- Physical-to-digital system mapping

### Challenges
- Translating spatial block placement into a reliable logical structure  
- Designing intuitive feedback for users  
- Representing nested structures (e.g., loops within conditionals)  
- Ensuring clarity without overwhelming beginners  

### Learning Outcomes
- Abstract programming logic can be modeled independently from text  
- Physical interfaces require very clear structural rules  
- Educational tools depend heavily on feedback and clarity  
- System architecture becomes critical when mapping real-world input to digital behavior
